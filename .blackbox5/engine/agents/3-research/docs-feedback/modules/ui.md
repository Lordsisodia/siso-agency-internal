# Module: UI Feedback (Docs Feedback Agent)

## Goal
Turn UI-related feedback into clear, testable fixes and a short prioritized list.

## Checklist
- [ ] Identify UI surfaces mentioned (page, component, flow)
- [ ] Translate vague feedback into an observable claim
- [ ] Determine root cause category:
  - IA/navigation
  - copy/labeling
  - layout/information density
  - state/empty states
  - affordances/CTA hierarchy
- [ ] Propose smallest viable fix first
- [ ] Define acceptance criteria (what “fixed” means)

## Output format
- Findings (bullets, each linked to a file path)
- Proposed fix (smallest first)
- Acceptance criteria
- Open questions (if any)
