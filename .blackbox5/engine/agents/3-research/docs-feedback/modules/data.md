# Module: Data / DB Simplicity (Docs Feedback Agent)

## Goal
Convert data-related feedback into concrete schema/logic simplifications and risk-aware actions.

## Checklist
- [ ] Identify whether issue is:
  - schema design / normalization
  - data integrity / constraints
  - query shape / performance
  - business logic ambiguity
- [ ] Determine what “truth” should be (source of truth)
- [ ] Identify safe simplification opportunities
- [ ] Note migration risk and rollout strategy

## Output format
- Problem statement + evidence
- Proposed simplification(s)
- Risks + mitigations
- Acceptance criteria
