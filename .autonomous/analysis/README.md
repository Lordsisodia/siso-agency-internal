# Analysis Directory

**Purpose:** Store analysis outputs, research findings, and investigation results.

---

## Directory Structure

```
analysis/
├── findings/       # Research findings
├── reports/        # Analysis reports
├── comparisons/    # Comparative analyses
└── README.md       # This file
```

---

## Usage

Place analysis outputs here when conducting:
- Codebase analysis
- Performance analysis
- Architecture reviews
- Technology comparisons
- Pattern detection

---

## File Naming Convention

```
{category}-{description}-{YYYYMMDD}.md
```

Examples:
- `codebase-auth-module-20260209.md`
- `performance-api-endpoints-20260209.md`
- `comparison-database-options-20260209.md`
