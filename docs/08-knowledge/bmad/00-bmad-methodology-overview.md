## What is BMAD?

BMAD is a **comprehensive methodology framework** that provides:

1. **Structured Development Process** - 4-phase methodology with enforced gates
2. **Specialized Agent System** - 12+ domain-expert AI agents with baked-in expertise
3. **Battle-Tested Workflows** - 50+ reusable YAML-based execution patterns
4. **Architecture Enforcement** - Automated domain-driven architecture validation
5. **Brownfield Expertise** - Specialized workflows for existing codebases

## Core Philosophy

**"Complexity lives in the system, not the user workflow"**

BMAD handles the complexity of multi-agent orchestration so developers can focus on building features.

```

PHASE 1: ELICITATION (Mary - Business Analyst)
├── Market research

```

## When to Use BMAD

**Perfect for:**
- Complex multi-domain projects
- Team-based development (2+ developers)
- Projects requiring architecture discipline
- Brownfield integrations
- Long-term maintenance projects

**Overkill for:**
- Single-file fixes
- Simple component updates
- Quick prototypes
- Solo dev with simple requirements

## BMAD vs GSD

**BMAD and GSD are complementary:**

| Aspect | BMAD | GSD |
|--------|------|-----|
| **Focus** | Methodology + Architecture | Context Engineering + Execution |
| **Team Size** | 2+ developers | Solo developers |
| **Process** | Structured phases | Atomic tasks |
| **Agents** | 12+ specialized | 11 generalist |
| **Strength** | Discipline, quality, scale | Speed, efficiency, solo dev |

**Best Practice:** Combine BMAD methodology with GSD execution patterns for optimal results.
